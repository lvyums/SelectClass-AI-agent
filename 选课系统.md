## 一、项目简介
### 1.1 项目名称
SelectClass — 智能选课系统（设计模式重构版）

### 1.2 项目定位
本系统是一款基于经典设计模式规范重构的学生选课管理平台，针对传统选课系统**耦合度高、代码冗余、可维护性差**等痛点，整合**创建型、结构型、行为型**三大类共9种主流设计模式进行架构重构，实现业务模块职责解耦、代码高度复用、功能快速扩展，搭建标准化、易维护、易迭代的前后端分离选课管理平台。

### 1.3 核心功能
| 功能模块 | 功能说明 |
| ---- | ---- |
| 登录认证 | 学生注册登录、JWT令牌认证、bcrypt密码加密存储 |
| 课程浏览 | 按院系、课程类型、难度筛选、关键词搜索、课程详情查看 |
| 选课退课 | 单门选课、批量选课、一键退课、自动课程时间冲突检测 |
| 智能对话选课 | 接入大模型实现自然语言指令选课、退课、查课 |
| 个性化课程推荐 | 依据学生专业、年级、兴趣标签智能匹配推荐课程 |
| 个人中心 | 个人信息编辑、专业年级兴趣信息维护 |
| 全局主题切换 | 支持亮色/暗色双模式页面主题切换 |

### 1.4 重构核心亮点
1. 落地**9种经典设计模式**，全覆盖三大设计模式分类，贴合软件开发设计思想
2. 前后端彻底分层解耦，后端遵循MVC架构，前端采用状态分层架构
3. 策略模式管控选课规则，不同课程类型独立校验逻辑，严格遵循开闭原则
4. 全局统一接口响应格式，大幅降低前后端联调开发成本
5. 权限拦截+日志记录双装饰器组合使用，横切业务逻辑统一管理

## 三、九大设计模式落地应用详解
### （一）创建型设计模式
#### 1. 单例模式（Singleton Pattern）
**应用场景**
1. 后端全局数据库连接实例、跨域配置实例
2. 后端项目全局环境配置类
3. 前端全局统一Axios请求实例
4. 前端Pinia全局状态仓库

**核心作用**
保证系统全局仅存在唯一实例对象，避免资源重复创建，节省服务器与客户端系统资源，统一全局配置与请求规范。

**核心应用代码**
后端数据库单例
```python
from flask_sqlalchemy import SQLAlchemy
from flask_cors import CORS
db = SQLAlchemy()
cors = CORS()
```
前端请求单例
```javascript
import axios from 'axios'
const request = axios.create({
  baseURL: '',
  timeout: 30000,
  headers: { 'Content-Type': 'application/json' }
})
export default request
```

#### 2. 工厂模式（Factory Pattern）
**应用场景**
1. 后端Flask项目统一应用创建工厂函数
2. 前端Vue项目全局应用初始化创建

**核心作用**
封装项目初始化全流程，整合配置加载、插件初始化、路由注册、数据库初始化等逻辑，将对象创建与业务使用彻底解耦，支持多开发环境快速切换。

**核心应用代码**
后端Flask应用工厂
```python
def create_app(config_name: str = "default") -> Flask:
    app = Flask(__name__)
    app.config.from_object(config[config_name])
    db.init_app(app)
    cors.init_app(app)
    _register_blueprints(app)
    with app.app_context():
        db.create_all()
    return app
```
前端Vue应用工厂
```javascript
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import router from './router'
import App from './App.vue'
const app = createApp(App)
app.use(createPinia())
app.use(router)
app.mount('#app')
```

### （二）结构型设计模式
#### 3. MVC架构模式
**应用场景**
贯穿前后端全项目核心分层架构，实现数据、视图、业务逻辑三层分离。

**分层职责划分**
1. 后端MVC
- Model层：models数据模型，定义数据库字段与实体关系
- View层：统一接口响应封装类，规范前端数据返回格式
- Controller层：蓝图路由层，负责请求接收、参数校验、调用业务服务

2. 前端MVC
- Model层：Pinia状态仓库，存储全局响应式数据
- View层：页面视图组件，负责页面UI渲染展示
- Controller层：组件事件+状态行为，调用接口处理前端业务逻辑

#### 4. 蓝图模式（Blueprint Pattern）
**应用场景**
1. 后端按业务模块拆分独立路由蓝图
2. 前端Vue Router模块化路由管理

**核心作用**
拆分臃肿路由代码，按业务域划分模块，实现路由解耦，统一批量注册管理，后期新增业务模块无需改动原有路由代码。

**后端业务蓝图划分**
| 蓝图名称 | 访问前缀 | 负责业务 |
| ---- | ---- | ---- |
| 认证蓝图 | /api/auth | 注册、登录、获取登录用户信息 |
| 课程蓝图 | /api/courses | 课程查询、课程详情 |
| 选课蓝图 | /api/selection | 选课、退课、已选课程查询 |
| 资料蓝图 | /api/profile | 个人信息修改维护 |
| 推荐蓝图 | /api/recommendations | 智能课程推荐 |
| 聊天蓝图 | /api/chat | 智能选课对话交互 |

#### 5. 代理模式（Proxy Pattern）
**应用场景**
1. 后端登录权限校验装饰器
2. 前端Axios请求拦截器（自动携带令牌）
3. 前端路由导航守卫（未登录拦截页面）

**核心作用**
不修改原有业务代码，通过代理拦截请求、页面访问、接口调用，统一完成权限校验、令牌注入、非法访问拦截。

#### 6. 装饰器模式（Decorator Pattern）
**应用场景**
后端全局接口请求日志记录装饰器

**核心作用**
动态为接口路由附加日志记录、耗时统计等通用功能，业务路由仅专注核心业务，通用横切逻辑统一通过装饰器实现，支持多装饰器叠加使用。

**组合使用**：权限代理装饰器 + 日志记录装饰器，先鉴权、后记录日志。

### （三）行为型设计模式
#### 7. 观察者模式（Observer Pattern）
**应用场景**
前端Pinia全局状态管理

**核心作用**
页面组件订阅全局状态数据，当状态数据发生修改后，所有关联组件自动同步刷新视图，摒弃传统组件层层传值、事件层层派发的繁琐写法。

#### 8. 模板方法模式
**应用场景**
后端统一接口响应模板封装、前端通用接口请求模板封装

**核心作用**
定义固定通用执行流程，统一返回数据格式、统一请求异常处理，子类/业务接口仅填充自有业务逻辑即可。

#### 9. 策略模式（Strategy Pattern）
**应用场景**
后端选课核心业务，区分**专业课、公共课、选修课**三类课程独立选课校验规则。

**核心作用**
将不同课程的选课限制规则封装为独立策略类，运行时根据课程类型自动匹配对应策略执行选课逻辑，消除大量if-else判断语句，新增课程类型仅新增策略类即可，符合开闭原则。

**选课策略规则**
1. 专业课：仅校验重复选课、时间冲突
2. 公共课：限制最大选课数量+基础冲突校验
3. 选修课：通用默认选课校验规则
